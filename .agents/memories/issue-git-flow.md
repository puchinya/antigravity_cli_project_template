# 🧠 Task Memory & Minutes: Issue - Define GitHub Flow Improved Branch Model

## 📋 イシュー概要
- **対象:** Gitブランチモデルおよびリリースフローの定義と `AGENTS.md` への反映
- **目的:** 開発の並行性とリリースの安定性を高めるため、「GitHub Flow 改善版（`main` をリリース用とし、各リリースバージョンごとに `dev/開発バージョン` ブランチを設ける構成）」を定義し、グローバルルールブック `AGENTS.md` に追記する。

## 🤝 決定事項 (Agreed Specifications)
- **ブランチ構成:**
  - `main`: リリース用（プロダクション安定版）。
  - `dev/開発バージョン`（例: `dev/v1.0.0`）: 各リリースバージョン用の統合・開発ブランチ。
  - `feature/issue-[番号]` または `fix/issue-[番号]`: 個別タスク用トピックブランチ（`dev/開発バージョン` から分岐、マージする）。
- **フロー:**
  - 各イシューは `dev/開発バージョン` からトピックブランチを切って開発し、テスト完了後に `dev/開発バージョン` にマージする。
  - リリースのタイミングで `dev/開発バージョン` を `main` にマージし、リリースタグを発行する。
